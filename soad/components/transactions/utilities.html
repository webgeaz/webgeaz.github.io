    Utilities in SOAD

    Rendering the View

    Import Statement:

    from utils import render

    1. Render Default View

    The render utility is used to display HTML views using the default layout, with optional logic processing via the SOAD framework. It supports rendering static views or dynamically populating views using context data (ctx) and Handlebars templates

    Basic Usage To redirect to a specific view (e.g., login), use:

    ctx.go_to = render.as_view(ctx, "login")

    This renders the login view without any additional logic or data.

    With Context Data To pass dynamic data to the view, populate ctx.output:

    ctx.output["name"] = "SOAD"
ctx.go_to = render.as_view(ctx, login)

    In this case, the name variable can be accessed in the template using Handlebars syntax ({{name}}).

    Rendering Page Directly from other group, and code

    To render a page directly from other <group>, and <code> bypassing the default layout:

    ctx.go_to = render.as_view(ctx, "register", "base", "reg")

    or you can do this:

    ctx.go_to = "%s/t/base/reg/register" % ctx.ctxPath

    Note

    • Use HTMX or context-based rendering (ctx) for dynamic content updates or logic processing.

    • The render.as_view() function ensures that the rendered page adheres to the default layout and theme structure used across the application.

    2. Render JSON Response

    This utility renders and returns a JSON response from the server, either from a specified object . It sets the appropriate content type and encoding for JSON responses and is typically used for API responses or HTMX requests expecting JSON data.

    Basic Usage

    To return a specific object as JSON instead of ctx.output as API, pass it as the second argument:

    data = {"status": "success", "user": "SOAD"}
ctx.go_to = render.as_json(ctx, data)

    This will return the data object as a JSON response.

    {
    "status": "success",
    "user": "SOAD"
}

    using toMaps() from List

    The toMaps() function is used to convert the list of objects to a list of maps. This is useful when you want to convert the list of objects to a list of maps for the JSON response.

    tb_user = Tb_user.findAll().toMaps() # findAll() (1) # toMaps() (2)

ctx.go_to = render.as_json(ctx, tb_user)
    1. findAll() is a method from activeJDBC which are to retrieve all the records from the database table in the form of a list of objects.
    2. toMaps() is a method from activeJDBC which are to convert the list of objects to a list of maps.

    Note

    • Return JSON data for frontend components using HTMX or JavaScript.

    • Create lightweight API responses within the same application logic flow.

    • Respond to AJAX requests or others plugin request with context or structured data.

    3. Render as HTML Response

    The render.as_html() utility is used to return a static HTML file directly to the client without any Handlebars or dynamic data rendering. This is typically useful for rendering pre-built HTML pages (such as documentation, help pages, or static content) stored in the server's file system.

    Basic Usage

    To render a static HTML file corresponding to the current request's code and location, use:

    ctx.go_to = render.as_html(ctx)

    This automatically infers the code and location from the current context using ctx.getCode() and constructs the full file path accordingly.

    With Custom Code and Location

    You can explicitly specify the code and location to render the static HTML file:

    ctx.go_to = render.as_html(ctx, "help", "/web/static/pages")

    This will render the HTML file located at <location>/<group>/<code>/<code>.html location. eg: /web/static/pages/<group>/help/help.html

    4. Render as String

    The render.as_string() utility is used to return a raw/simple HTML string as the response, bypassing templates and static files. This is ideal for rendering simple HTML snippets directly from code, such as inline alerts, messages, or dynamically generated fragments.

    Basic Usage

    To return a raw/simple HTML string as the response:

    ctx.go_to = render.as_string(ctx, "<h1>Access Denied</h1><p>You are not authorized.</p>")

    This sends the HTML string directly to the client browser without any further processing.

    4. Render as File

    The render.as_file() utility allows you to serve files (such as PDFs, images, or documents) to the client. It sets the appropriate headers and streams the file content for download or inline display in the browser.

    File types:
    • pdf: application/pdf.
    • image: image/png, image/jpeg.
    • text: text/plain, text/html, text/css, text/javascript.
    • documents: application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document, application/vnd.ms-excel, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet, application/vnd.ms-powerpoint, application/vnd.openxmlformats-officedocument.presentationml.presentation.

    Basic Usage

    ctx.go_to = render.as_file(ctx, "/path/to/report.pdf", "application/pdf")

    This will serve and download the specified PDF file to the client.

    Optional Parameters

    • filename: Custom filename for the download (defaults to the file’s name).
    • attachment: Set to False to display the file inline (e.g., open in-browser), instead of downloading.
    ctx.go_to = render.as_file(ctx, "/path/to/image.png", "image/png", filename="preview.png", attachment=False)

    This will serve the image file with a custom filename and display it inline in the browser.

    Note

    • Use render.as_file() for serving downloadable files or inline content.
    • Specify the correct MIME type for the file being served to ensure proper handling by the browser.
    • Use filename and attachment parameters to control the download behavior and filename.

    5. Render as blob

    The render.as_blob() utility is used to return binary data (such as images, audio, or other non-text files) as a response. It sets the appropriate headers and encodes the binary data for transmission over HTTP.

    Basic Usage

    To send a blob as a downloadable file:

    ctx.go_to = render.as_blob(ctx, content, "application/pdf", "invoice.pdf", attachment=True)

    To show the content inline in the browser:

    ctx.go_to = render.as_blob(ctx, content, "application/pdf", "preview.pdf", attachment=False)

    Note

    • content: The raw binary data (e.g., byte[] or bytes).

    • content_type: MIME type (e.g., "application/pdf", "image/png").

    • filename: The file name to be shown in the browser or used for downloading.

    • attachment: If True, prompts a download; if False, displays inline.

    6. Render as PDF

    The render.as_pdf() utility renders an HTML template (using Handlebars) into a PDF file. It supports dynamic data injection using ctx.output and can be downloaded or displayed inline.

    Basic Usage

    To render and download a PDF in HTML format(including the data):

    ctx.output["name"] = "SOAD"
ctx.go_to = render.as_pdf(ctx, view="invoice", filename="invoice.pdf", attachment=True)

    Or previe the PDF inline in the browser:

    ctx.output["name"] = "SOAD"
ctx.go_to = render.as_pdf(ctx, view="summary", filename="summary.pdf", attachment=False)

    Templates Rendering with Handlebars:

    The template supports dynamic content using ctx.output data:

    invoice.html
    <html>
    <head>
        <title>Invoice</title>
    </head>
    <body>
        <h1>Invoice for {{name}}</h1>
        <p>Thank you for your purchase!</p>
    </body>
</html>

    All Handlebars helpers like eq, and, dateFmt, ref_lookup, etc. are supported in the PDF.

    File Upload

    SOAD provides an easy way to upload files to the server. Usually the file will be uploaded to the server directory and the file path will be saved to the database for access usage.

    Import Statement:

    from utils import fileupload
    File types:
    • pdf: application/pdf.
    • image: image/png, image/jpeg.
    • text: text/plain, text/html, text/css, text/javascript.
    • documents: application/msword, application/vnd.openxmlformats-officedocument.wordprocessingml.document, application/vnd.ms-excel, application/vnd.openxmlformats-officedocument.spreadsheetml.sheet, application/vnd.ms-powerpoint, application/vnd.openxmlformats-officedocument.presentationml.presentation.

    To handle the file from the request from the user, which are to upload the file to the server. This are the ways to handle it:

    The uploaded File need to be in MultiPart form data
    • In the html the in the form tag, need to add this enctype="multipart/form-data" to upload the file to the server.
    index.html
    <form action="{{ctxPath}}/t/base/upload/upload_image" method="post" enctype="multipart/form-data">
    <!-- Your basic form here -->
     <input type="file" name="image" id="image" accept="image/png, image/jpeg, image/jpg">
</form>

    request = ctx.getRequest()
image = request.getParameter("image")       # File content (bytes)
image_fn = request.getParameter("image_fn") # Original File names <= Will be generated
image_ft = request.getParameter("image_ft") # MIME/file type <= Will be generated

    1. To Upload the File

    The write_file() method is used to save the uploaded file to the server directory.

    Usage Example:

    from java.io import File

file_location = "/path-to-upload
uploaded_file = File(file_location)

if not uploaded_file.exists(): # Check if the directory exists
    uploaded_file.mkdir()

if image and image_fn: # Check if the file is not empty
    file_path = uploaded_file.getPath() + File.separator + "image_" + image_fn # File path to save the file

    fn = File(file_path)

    fileupload.write_file(image, fn) # Write the file to the server directory

    Tip

    • The directory/path of the file, file name, and file type need to be save to the database for access usage.

    Rendering and Sending the Email

    SOAD provides an easy way to send the email to the user. The mailer class provides the methods to send the email to the user. but, you need to import the mailer class from the utils module.

    from utils import mailer

    mailer.send(sender_email, receiver_email, subject, email_content, html=None)

    There are two ways to write and send the email to the user:

    1. Using Simple String Methods:

    The below code is used to send the email to the user using the simple string methods.

    email_content = """
<table>
    <tr>
        <td>Hi, User</td>
    </tr>
    <tr>
        <td>Your OTP is: 1234</td>
    </tr>
</table>
"""

mailer.send("noreply@soad.online", "receiver@yopmail.com", "Your Subject Here", email_content, html=True)

    2. Using HTML File:

    The below code is used to send the email to the user using the HTML file.

    from utils import render

email_content = render.as_view(ctx, "email_template")()
mailer.send("noreply@soad.online", "receiver@yopmail.com", "Your Subject Here", email_content, html=True)

    PDF Generator

    SOAD provides a simplified and robust way to generate PDF files directly from HTML templates or raw HTML strings. With the pdf utility, you can render dynamic content into a well-formatted PDF and either stream it or save it to the file manager within the SOAD environment.

    Import Statement:

    from utils import pdf

    1. Generate PDF from HTML

    You can render a dynamic template using render.as_view(...) and convert it directly into a PDF file:

    Usage Example:

    pdf_file = "/path-to-save-pdf-file"
content = render.as_view(ctx, "invoice")()

pdf.generate(content, pdf_file)

    2. Generate PDF from HTML String:

    If you prefer working with raw HTML strings (e.g., generated manually or from external data), you can still produce a PDF easily:

    Usage Example:

    pdf_file = "/path-to-save-pdf-file"
content = """
    <html>
        <body>
            <h1>Your Invoice</h1>
            <p>Thank you for your purchase!</p>
        </body>
    </html>
    """
pdf.generate(content, pdf_file)

    Note

    The path for saving pdf file should be defined if want to save the pdf file to the server
