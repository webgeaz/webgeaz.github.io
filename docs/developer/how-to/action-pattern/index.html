    Action Pattern¶

    SUFIA is a flexible framework that allow you to build your application in your own styles. SUFIA only dictates certain convention and the rest is open for you to design your application.

    Despite the flexibilities, sometimes it can become quite challenging for developer who are relatively new in web application development to adapt with SUFIA methodologies.

    For JAVA developers who familiar with action based framework (i.e. Struts, Stripes, Spring MVC, etc), one common pattern that is available in the "method based action". Using this approach, you will design the application based on the Action classes and separate the process based on methods inside the Action class.

    To implement the same pattern in SUFIA is quite simple, you need to design your Transaction using these styles:

    Form Template

    <form class="form-horizontal" 
    action="{{ctxPath}}/t/HR/report?invoke" 
    method="post">
  <div class="form-group">
      <label class="control-label col-sm-2">Title</label>
      <div class="col-sm-8">
          <input type="text" name="title" class="form-control">
      </div>
  </div>
  <div class="form-group">
      <label class="control-label col-sm-2">Applicant</label>
      <div class="col-sm-8">
          <textarea name="applicant" class="form-control" rows="2">
          </textarea>
      </div>
  </div>
  <div class="form-group">
      <label class="control-label col-sm-2">Description</label>
      <div class="col-sm-8">
          <textarea name="description" class="form-control" rows="2">
          </textarea>
      </div>
  </div>
  <div class="form-group">
      <div class="col-sm-8 col-sm-offset-2">
          <button type="submit" name="d_action" value="save_report" 
            class="btn btn-primary">
              Save
          </button>
      </div>
  </div>
</form>

    Post-Processor

    import Report

if d_action == "save_report":
    #do processing...

if d_action == "delete_report":
    #another action...

    By using this simple scheme, you can keep your code clean and easy to navigate.
