    Customization of User Management and Login Process¶

    Custom User¶

    Sometimes, there is a need to have your own implementation of user profiles. Especially if there is a need for more details which default SUFIA User did not provide.

    There is two approach to achieve this, one is by implementing the whole set of User lifecycle. This means you need to create your own user, menu, role etc. based on the requirement. This is quite straight forward.

    However, if you still want to make use of SUFIA built-in User, Role, Menu functions you need to interact with SUFIA API. In order to linked it back to SUFIA User table, you need to use UserService.

    Example:

    Consider we created our own user table like below: 

    my_user
-------
id
login_id
user_id
name
email
created_date

    Backend Code: 

    import My_user
from java.util import Date
from sufia import User

today = Date()

userService = service.get("UserService")
roleService = service.get("RoleService")
menuService = service.get("MenuService")
my_role = roleService.findByName("My Role")
my_menu = menuService.findByName("My Menu")

if d_action == "save_user":
    my_user = My_user.first("login_id=?", login_id)
    if my_user:
        err = "User already exists"
    else:
        sufia_user = userService.findByLoginId(login_id)
        if sufia_user:
            err = "User already exists"
        else:
            sufia_user = User()
            sufia_user.set("loginId", login_id)
            sufia_user.set("name", name)
            sufia_user.set("password", password)
            sufia_user.set("email", email)
            sufia_user = userService.register(sufia_user, [int(my_role.get("id"))], int(my_menu.get("id")), webCtx.getCurrentScope())
            if sufia_user:
                # only create My_user if SUFIA User creation successful
                my_user = My_user()
                my_user.set("login_id", login_id)
                my_user.set("user_id", sufia_user.get("id")) # link with SUFIA User's id
                my_user.set("name", name)
                my_user.set("email", email)
                my_user.set("created_date", today)
                my_user.saveIt()
                go_to = "%s/t/list/user" % (ctxPath)

    Custom Login¶

    There is two ways available to implement your own login process based on your requirement.

    (Option 1) Using Transaction with LoginService¶

    (Option 2) Override Login process¶

    SUFIA provides a way to override default login process. You can create your own Java class which implement LoginService. After that you need to set your custome class using properties login.service on sufia.properties configuration file.

    This is useful if you want to implement different logic such as login using Single Sign-On (SSO), Active Directory / LDAP or other external database / login provider.

    Example (Login without using password):

    package webgeaz.sufia.service;

import com.google.inject.Inject;
import webgeaz.sufia.model.UserProfile;
import webgeaz.sufia.web.mgr.UserManager;
import webgeaz.sufia.web.service.LoginService;

public class LoginWithoutPassword implements LoginService {

    private final UserManager userManager;

    @Inject
    public LoginWithoutPassword(UserManager userManager) {
        this.userManager = userManager;
    }

    @Override
    public UserProfile login(String loginId, String password) {
        if (loginId != null && loginId.startsWith("VALID-")) {
            return userManager.findByLoginId(loginId);
        } else {
            return userManager.login(loginId, password);
        }
    }

}
